# 计费周期计算示例

## 月末日期处理规则

当首次付款日期是月末时，系统会智能处理续费日期，确保每个月都使用该月的最后一天（如果原日期超出该月天数）。

### 示例 1：1月31日开始的月度订阅

| 首次付款 | 第2次   | 第3次   | 第4次   | 第5次   | 第6次   |
| -------- | ------- | ------- | ------- | ------- | ------- |
| 1月31日  | 2月28日 | 3月31日 | 4月30日 | 5月31日 | 6月30日 |

**说明**：

- 2月只有28天（非闰年），所以使用2月28日
- 3月有31天，恢复到31日
- 4月只有30天，使用4月30日
- 5月有31天，恢复到31日
- 6月只有30天，使用6月30日

### 示例 2：1月31日开始的月度订阅（闰年）

| 首次付款      | 第2次         | 第3次         | 第4次         |
| ------------- | ------------- | ------------- | ------------- |
| 2024年1月31日 | 2024年2月29日 | 2024年3月31日 | 2024年4月30日 |

**说明**：

- 2024年是闰年，2月有29天，所以使用2月29日

### 示例 3：1月30日开始的月度订阅

| 首次付款 | 第2次   | 第3次   | 第4次   |
| -------- | ------- | ------- | ------- |
| 1月30日  | 2月28日 | 3月30日 | 4月30日 |

**说明**：

- 2月只有28天，使用2月28日
- 其他月份都有30天或更多，保持30日

### 示例 4：1月29日开始的月度订阅

| 首次付款 | 第2次   | 第3次   | 第4次   |
| -------- | ------- | ------- | ------- |
| 1月29日  | 2月28日 | 3月29日 | 4月29日 |

**说明**：

- 2月只有28天（非闰年），使用2月28日
- 其他月份都保持29日

### 示例 5：2月29日开始的年度订阅（闰年）

| 首次付款      | 第2次         | 第3次         | 第4次         |
| ------------- | ------------- | ------------- | ------------- |
| 2024年2月29日 | 2025年2月28日 | 2026年2月28日 | 2027年2月28日 |

**说明**：

- 2024年是闰年，有2月29日
- 2025、2026、2027年都不是闰年，使用2月28日
- 2028年是闰年，会恢复到2月29日

## 其他计费周期

### 每周订阅

- 简单地加7天，不受月末影响
- 例如：1月26日 → 2月2日 → 2月9日

### 每日订阅

- 简单地加1天，不受月末影响
- 例如：1月31日 → 2月1日 → 2月2日

### 每年订阅

- 保持相同的月和日
- 如果目标年份该月没有该日期，使用该月最后一天
- 例如：2024年2月29日 → 2025年2月28日

## 实现逻辑

```swift
// 核心算法
let targetDay = min(originalDay, maxDaysInTargetMonth)
```

这个简单的逻辑确保：

1. 如果目标月份有足够的天数，保持原日期
2. 如果目标月份天数不足，使用该月最后一天

## 测试用例

### 月度订阅测试

- ✅ 1月31日 + 1个月 = 2月28日（非闰年）
- ✅ 1月31日 + 1个月 = 2月29日（闰年）
- ✅ 1月31日 + 2个月 = 3月31日
- ✅ 3月31日 + 1个月 = 4月30日
- ✅ 5月31日 + 1个月 = 6月30日

### 年度订阅测试

- ✅ 2024年2月29日 + 1年 = 2025年2月28日
- ✅ 2024年2月29日 + 4年 = 2028年2月29日
- ✅ 2023年1月31日 + 1年 = 2024年1月31日

### 边界情况

- ✅ 12月31日 + 1个月 = 1月31日（跨年）
- ✅ 12月31日 + 2个月 = 2月28日（跨年 + 月末）
- ✅ 11月30日 + 1个月 = 12月30日
